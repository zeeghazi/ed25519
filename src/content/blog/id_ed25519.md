If you have generated an SSH key, you may have spotted two files named
`id_ed25519` and `id_ed25519.pub` in your `~/.ssh` folder. Here is exactly what
they are.

## id_ed25519 vs id_ed25519.pub

`id_ed25519` is your **private** Ed25519 SSH key — the secret that proves your
identity. `id_ed25519.pub` is the matching **public** key, which you hand out
to servers. They are two halves of one key pair.

| File             | What it is                      | Share it?   |
| ---------------- | ------------------------------- | ----------- |
| `id_ed25519`     | Private key — keep secret       | **Never**   |
| `id_ed25519.pub` | Public key — install on servers | Yes, freely |

The name comes from a convention: `id_` + the key type. An RSA key would be
`id_rsa` / `id_rsa.pub`; an Ed25519 key is `id_ed25519` / `id_ed25519.pub`. Both
live in `~/.ssh/` by default (`%USERPROFILE%\.ssh\` on Windows).

## How to create id_ed25519

Run `ssh-keygen` with the Ed25519 key type:

```
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Accept the default location and set a passphrase. That writes both files:

```
~/.ssh/id_ed25519        # private key
~/.ssh/id_ed25519.pub    # public key
```

The full walkthrough — adding the key to `ssh-agent` and installing it on
GitHub or a server — is in our [Ed25519 SSH key guide](/ed25519-ssh-key/).

## What is inside each file

`id_ed25519.pub` is a single line you can safely print:

```
cat ~/.ssh/id_ed25519.pub
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... your_email@example.com
```

`id_ed25519` is the private key, wrapped in an
`-----BEGIN OPENSSH PRIVATE KEY-----` block and (if you set one) encrypted with
your passphrase. **Do not** print or paste this file anywhere.

## File permissions matter

SSH refuses to use a private key that other users can read. Keep the
permissions tight:

```
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

## Why Ed25519 instead of RSA?

`id_ed25519` files are tiny and fast compared to `id_rsa`, with strong modern
security. See [Ed25519 vs RSA](/blog/ed25519-vs-rsa/) for the comparison, or
[what is Ed25519](/blog/what-is-ed25519/) for the fundamentals.
